Getting Started for Developers

Tyrian provides front-end developers & engineers a collection of reusable HTML and CSS elements to build websites and user interfaces.

Tyrian's CSS

The system uses Bootstrap 5.x as the foundation for its HTML and CSS. Adopting Tyrian enables developers to use consistent markup, styles, and behavior in prototype and production work.

Code styles

  • Syntax: rules and structure for formatting our code (e.g. curly braces always go on a new line)
  • Semantics: rules for making our code understandable (e.g. alphabetizing property declarations)

Don't get caught up in wars over tabs vs spaces. These styles help us produce consistently written code, not to achieve theoretical perfection.

BEM

Tyrian uses BEM which is an abbreviation for the methodology that we follow – Block, Element, and Modifier.

Block

Standalone entity that is meaningful on its own.

Examples

header

container

nav

checkbox

label

input

Element

A part of a block that has no standalone meaning and is semantically tied to its block.

Examples

nav item

list item

checkbox description

card body

Modifier

A flag on a block or element. Use them to change appearance or behavior.

Examples

disabled

highlighted

checked

fixed

size

big

color yellow

To learn more about BEM visit getbem.com/introduction
