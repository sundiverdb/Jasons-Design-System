FAQ

Commonly asked questions about the design system.

What is Tyrian?

Tyrian is HealthEquity's Design System which includes a collection of individual brand, foundation, components, mobile guidelines, and patterns for creating a unified frontend experience across HealthEquity's products. Tyrian is built on top of Bootstrap 4.

How is this any different from a style guide?

A style guide is a single point in the design process. A design system is a living document that is continually refined to serve an entire ecosystem of products.

How can I participate in building the design system?

We have some rough guidelines for contributing, chat with a member of the UX team or attend a UX Guild meeting to find out how you could participate. We appreciate all the input and help we can get.

Which browsers do we support?

We hope to be able to support the following browsers:

  • Chrome Latest
  • Edge Latest
  • Firefox Latest
  • Safari Latest

I just found a bug, who should I tell?

Contact the UX team/UX guild directly, as we don't have a process in place for handling bugs, yet!

Why hasn't the bug I reported been fixed yet?

Have you made a big enough stink about it? If not, please bring it back up to our attention. We will try and address any bugs that are found as soon as possible.

I can't update to the latest version of a component, because it will break stuff?

Please let us know of the issue so we can help you figure out a solution.

I can't seem to find the component I need?

If we haven't provided a component here in the design system yet we are most likely using the Bootstrap default. If the component you are looking for doesn't have a bootstrap version, let us know.

Where do I go if I still have some concerns or questions?

Please contact a representative of the UX team/UX guild.
