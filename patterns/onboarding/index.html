Onboarding

We need to provide the user with the necessary knowledge, skills, and education to achieve their goals.

Users want to get started using an application right away but but might need guidance with the look, feel, and functions once filled with data. Onboarding is a designed series of interactions and instructions that help ease the user into the product experience.

Onboarding should give the user a sense of what will be required to get what they need out of the product. First impressions only happen once and can dictate whether your users will continue to use your product or not.

A good empty state is critical for educating and providing a good first time experience. Consider presenting sample content or educating on what the user can do with in the application. If the screen is simply left blank the user might feel intimidated if there is little or no guidance on what to do.

Types of onboarding #

  • Quick Tour
  • Guided Tour
  • Inline Hints
  • Guided Task Completion
Quick Tour

This generally appears on the first launch of the application, the user is presented with a few quick screens outlining the value of the product and some basic information on how to get around.

This simple introduction serves as a welcome mat for users. It should never contain more than a few short phrases accompanied by simple graphics.

Guided Tour

This method also usually appears on the first launch, the user is presented with a series of popovers that draw attention to areas of interaction, button, navigation menus and help alert the user to useful elements in the app.

These popovers are especially useful in highlighting more complex interfaces where items might not be immediately apparent or easy to understand. Guide users to one action or element at a time and keep the explanations simple and concise, avoid highlighting more than 6-7 elements.

Inline Hints

Help the users with just in time guidance that educates new or unfamiliar features in an unobtrusive way.

These hints might be delivered through a variety of messaging components, such as tooltips, popovers, alerts, and notifications. These should be used when new features or changes occur to the product and might also provide a button to access more extensive oboarding patterns like a quick or guided tour.

Guided Task Completion

Sometimes the best way to learn is by doing. guided task completion are prompts for users to follow a series steps. This can give users an opportunity to familiarize themselve with the product staight away. By helping the users complete a task you are also improving the chance that they will continue to use the feature or product.
