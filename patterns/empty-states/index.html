Empty States

Empty states occur when an item's content can't be shown.

Empty states can display a wide variety of content. For example they can include a list without items, or a search that returns no results. Although not typical they need to be designed to prevent confusion.

Empty states can be used to provide educational content to help with onboarding, it can build trust, and encourage continued use of a product. Along with generating awareness and building the brand.

Types of empty states

  • First use occurs within a new product or service when their is nothing to show.
  • User cleared occurs when users complete actions and the result is an empty screen.
  • Errors these occur when something goes wrong, or when there are issues with network connection.
  • No results or data occurs when there is nothing to show. This can happen if someone performs a search and the query is empty or there isn't any data to show after filtering the content.

Displaying an empty state

The most basic empty state consists of a non-interactive text tagline, but may include an image or icon when desired.

Use a tagline that:
  • Has a clear and helpful message
  • Is consistent with HealthEquity's brand and language
  • Conveys the purpose of the screen, without appearing actionable
Use an image that:
  • Has a neutral or professional tone
  • Is consistent with HealthEquity's brand and tone

Simple Empty State #

  1. Empty message header clearly informs that the search query did not return any results.
  2. Description or instruction should provide a helpful reference and let the users know what is going on.

Illustrated Empty State #

  1. Icon/image that helps inform about the current empty state and is builds on the brand.
  2. Header title should explain the general context of the empty state.
  3. Description or instruction should provide a helpful reference and let the users know what is going on.
  4. Button/link should give the user an action that they may take next.
