Patterns

Page Header

The page header is fixed to the uppermost portion of the webpage or product.

The page header contains global elements such as logos, navigation, and primary system actions and information.

Default Header

  1. The page header spans the full width of the view port and is fixed to the top of the page.
  2. Logos are restricted to a max-width of 240px and a max-height of 48px. Logos smaller than this will render at size. This allows our current suggested size of 360px by 64px for partner logos to be proportionally reduced in size with accuracy.
  3. Navigation hover menu text summarizes the content of the links in the hover menu, and are indicated by an angle down icon to the right of the navigation text.
  4. Navigation links summarize the content of the page they link to.
  5. The username and id number appears below the right side of the page header. This element is global to inform the user of their session and id number.
  6. System actions and information quick event driven actions or links to key information for the account appears right aligned on the page header with an associated icon and a summarized title.

Hover States #

  1. A hover menu navigation link displays a dropdown menu with additional navigation links on hover.
  2. Dropdown hover menu links appear with a white background.
  3. Dropdown hover menu links on hover appear with a light gray background.
  4. All primary navigation links on hover have purple text and a purple border on the bottom.
  5. The icons and labels that are in the system action area turn purple on hover.

Dropdowns #

  1. A dropdown can be added to one of the links the right aligned system actions menu instead of executing the event directly.

Co-Branding #

  1. The partner logo replaces HealthEquity's when requested.
  2. The stacked HealthEquity logo will be added to the far right side of the page header by default, but a partner may request that the HealthEquity logo does not show here and only shows in the footer.

Responsive #

  1. At a tablet size viewport less than 768px wide the page header navigation links should not display. The navigation links display depends on their width and may collapse at a wider viewport breakpoint.
  2. The hamburger menu icon appears to the left of the logo once the navigation links have collapsed within.
  3. The logo at tablet size remains at either a max-width of 240px or a max-height of 48px.
  4. A mobile size viewport can be used all the way to a minimum of 320px wide.
  5. The logo at a mobile viewport of 480px or less the logo is restriced to a max-width of 120px.

Responsive Menu

  1. The responsive menu slides out as a drawer on the left side of the viewport offsetting a pages content when the hamburger menu icon is selected. The menu also displays the full height of the viewport and becomes scrollable when necessary.
  2. The navigation links have a similar appearance to those of the hover menu with a white background and gray border on the bottom.
