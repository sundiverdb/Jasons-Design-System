Components

Buttons

Custom button styles for actions in forms, dialogs, and more with support for multiple sizes, states, and more.

Examples

Using Bootstrap we have several predefined button styles, each serving its own semantic purpose, with a few extras thrown in for more control.

<button type="button" class="btn btn-primary">Primary</button>
<button type="button" class="btn btn-secondary">Secondary</button>
<button type="button" class="btn btn-neutral">Neutral</button>

<button type="button" class="btn btn-link btn-action"><span class="t-bomb"></span> Action Link</button>
<button type="button" class="btn btn-link"><span class="t-link"></span> Link</button>

Disabled State

Make buttons look inactive by adding the disabled boolean attribute to any <button> element.

Try and design without using disabled states. Disabled elements may cause confusion when the reason for the element being disabled isn't clear. Provide a clear validation reason when disabling an element.

<button type="button" class="btn btn-primary" disabled>Primary</button>
<button type="button" class="btn btn-secondary" disabled>Secondary</button>
<button type="button" class="btn btn-neutral" disabled>Neutral</button>

<button type="button" class="btn btn-link btn-action" disabled><span class="t-bomb"></span> Action Link</button>
<button type="button" class="btn btn-link" disabled><span class="t-link"></span> Link</button>

Sizes #

Buttons by default have a minimum width of 5rem or 80px, to override this for smaller buttons add the .btn-tight  class for a button without a minimum width and tightened padding.

<button type="button" class="btn btn-primary">Home</button>
<button type="button" class="btn btn-primary btn-tight">Home</button>

Need a larger or smaller button? Add .btn-lg or .btn-sm  for additional sizes.

<button type="button" class="btn btn-primary btn-lg">Large button</button>
<button type="button" class="btn btn-secondary btn-lg">Large button</button>
<button type="button" class="btn btn-primary btn-sm">Small button</button>
<button type="button" class="btn btn-secondary btn-sm">Small button</button>

Create block level buttons–those that span the full width of a parent–by adding .btn-block.

Block level buttons are usually reserved for narrower cards where it makes sense that the button would span the entire width.

<button type="button" class="btn btn-primary btn-lg btn-block">Block level button</button>
<button type="button" class="btn btn-secondary btn-lg btn-block">Block level button</button>

Button Groups #

Wrap a series of buttons with .btn in .btn-group

<div class="btn-group" role="group" aria-label="Basic example">
  <button type="button" class="btn btn-primary">Left</button>
  <button type="button" class="btn btn-secondary">Middle</button>
  <button type="button" class="btn btn-neutral">Right</button>
</div>

Recommendations #

  • Make sure the purpose and state of the button are clear to the user.
  • Present only one or two buttons to the user at a time, for example, Accept and Cancel. If you need to expose more options to the user consider a different approach with a single command button to trigger the actions.
  • For an action that needs to be available across multiple pages with the portal or app, consider using a summary section or toolbar that is persistent rather than duplicating buttons.
  • When using link buttons consider using an icon in front to clarify the action being taken.

Positioning #

Button positioning is determined by the container that the button(s) are within.

Do

On pages that contain a single button, the button should be left-aligned with the rest of the pages content.

On pages that contain two buttons, buttons should be left-aligned with the primary button being place first on the left.

Modal dialogs or cards with a single button should right-align the button. This button should perform a safe, non-destructive action.

Modal dialogs or cards with two buttons should be right-aligned. With the primary button on the right.
 

Don't

On pages that contain buttons, the button(s) should not be right-aligned.

On pages that contain two buttons, the buttons should not swap positions with the secondary button appearing first.

Buttons within a modal dialog or card should not be left-aligned.
 

Dialogs with two buttons should not have the primary button on the left.
