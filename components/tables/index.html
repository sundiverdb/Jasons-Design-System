Tables

Use tables to display data sets, that provide consistent comparable, readable, scannable, and sortable views of the data

Example

Due to the widespread use of tables across third-party widgets like calendars and date pickers, we've designed our tables to be opt-in. Just add the base class .table to any <table>, then extend with custom styles or our various included modifier classes.

Tables are created using the same classes and structures provided in Bootstrap. They have been branded and modified for HealthEquity.

Responsive CSS Tables #

These tables appear to be HTML tables in large form factor but in small form factor they become cards.

It is recommended to use an unordered list but the styles are related to classes not tags so different markup structure can be used.

Currency

Always right align the titles and content of table columns that pertain to currency. Add the .text-right class from Boostrap's typography utilities where necessary to right align content.

Table View

  • <table> = .css-table
  • <thead> = .css-thead
  • <tbody> = .css-tbody
  • <tfoot> = .css-tfoot
  • <tr> = .css-tr
  • <th> = .css-th
  • <td> = .css-td

Card View

Place inline labels inside span tags with the .css-td class and use .d-none-lg and .d-none-xl classes to hide the lable for large form factor when in the table grid view.

  • Header for card with large type = .tcard-header
  • Elements that go in the footer of a card = .tcard-footer
