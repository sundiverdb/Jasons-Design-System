Progress

The progress component visually shows the status of a specific process, either as a single event or through a set of steps.

How it works

Progress components are built with two HTML elements, some CSS to set the width, and a few attributes. We don't use the HTML5 <progress> element, ensuring you can stack progress bars, animate, and place text labels over them.

  • We use the .progress as a wrapper to indicate the max value of the progress bar.
  • We use the inner .progress-bar to indicate the progress so far.
  • The .progress-bar requires an inline style, utility class, or custom CSS to set their width.
  • The .progress-bar also requires some role and aria attributes to make it accessible.

Example

<div class="progress">
  <div class="progress-bar" role="progressbar" aria-valuenow="0" aria-valuemin="0" aria-valuemax="100"></div>
</div>
<div class="progress">
  <div class="progress-bar" role="progressbar" style="width: 25%" aria-valuenow="25" aria-valuemin="0" aria-valuemax="100"></div>
</div>

Bootstrap provides a handful of utilities for setting width. Depending on your needs, these may help with quickly configuring progress.

<div class="progress">
  <div class="progress-bar w-75" role="progressbar" aria-valuenow="75" aria-valuemin="0" aria-valuemax="100"></div>
</div>

Labels #

Add labels above your progress bars by placing a label tag just before the .progress div.

$250.00 of $500.00
$375.00 of $500.00

  <div class="progress-bar w-25" role="progressbar" aria-valuenow="25" aria-valuemin="0" aria-valuemax="100"></div>

Multiple Bars #

Include multiple progress bars in a progress component if you need. Use a border to separate colors that might have similar contrast.

<div class="progress">
  <div class="progress-bar bg-lavender chart-border-right" role="progressbar" style="width: 15%" aria-valuenow="15" aria-valuemin="0" aria-valuemax="100"></div>
  <div class="progress-bar bg-capri chart-border-right" role="progressbar" style="width: 30%" aria-valuenow="30" aria-valuemin="0" aria-valuemax="100"></div>
  <div class="progress-bar bg-mint chart-border-right" role="progressbar" style="width: 24%" aria-valuenow="20" aria-valuemin="0" aria-valuemax="100"></div>
</div>

Stepped Progress #

Use <div> tags. Each step is a new <div> tag. Completed steps are marked with the class "complete" on the <div> tag. The first step will always be marked complete so the .complete class is not necessary on that step.

There should never be more than five to six steps maximum when using the stepped progress component.

    <span class="progress-step-bubble">1</span>
  </div>
  <div class="progress-step" role="progressstep" aria-valuenow="2" aria-valuemin="2" aria-valuemax="4">
    <span class="progress-step-bubble complete">2</span>
  </div>
  <div class="progress-step" role="progressstep" aria-valuenow="2" aria-valuemin="3" aria-valuemax="4">
    <span class="progress-step-bubble">3</span>
  </div>
  <div class="progress-step" role="progressstep" aria-valuenow="2" aria-valuemin="4" aria-valuemax="4">
    <span class="progress-step-bubble">4</span>
